# 출판사 홈페이지 구축 PRD (Vite 기반)

- 출판사명: 도서출판 동틀녘 
- 문서 버전: v1.1  
- 작성일: 2025-12-19 (Asia/Seoul)  
- 대상: 1인 편집자 운영 / 외부서점 링크 판매 모델

---

## 1. 배경과 목적

### 1.1 배경
출판사 웹사이트를 구축한 뒤, 운영 중에 **신간 발매 소식/공지/칼럼** 등을 꾸준히 게시하고, **도서 목록/저자 페이지를 구조적으로 분리**하여 관리하고자 함. 결제는 자체 처리하지 않고 **외부서점 링크로 유도**함.

### 1.2 목표(Goals)
1. 출판사의 신뢰도·브랜딩 강화(소개/미션/연혁/연락처)
2. **도서(전자책 포함) 목록 + 상세 페이지** 제공 및 외부서점 구매 링크 유도
3. **저자 목록 + 저자 상세 페이지** 제공 (저자별 도서 자동 연결)
4. **소식(신간/공지/이벤트)** 게시 및 공유(OG 이미지/SEO) 최적화
5. 1인 운영자가 개발 지식 없이도 **CMS 관리자 화면에서 콘텐츠 등록/수정/발행** 가능

### 1.3 비목표(Non-goals)
- 자체 결제/장바구니/회원가입/포인트/배송/재고 관리
- 댓글/커뮤니티(초기 범위에서는 제외)
- 복잡한 편집 워크플로(다단계 승인) — 1인 운영 기준

---

## 2. 핵심 원칙(Principles)

- **운영 편의성 우선:** “관리자 페이지를 직접 개발” 대신 **Git 기반 CMS(Keystatic)의 관리자 UI 사용** - *비용 0원, 서버 불필요, 뛰어난 DX*
- **정적 배포 친화:** Vite 빌드 산출물로 배포 가능한 구조(캐시/성능 유리)
- **SEO/공유 품질:** 도서/저자/소식 상세는 검색·공유에서 내용이 잘 보이도록 설계
- **확장 가능:** 향후 뉴스레터/구독/검색 고도화 등 기능 확장 여지 확보

## 3. 이해관계자 및 사용자

### 3.1 이해관계자(Stakeholders)
- 운영자(편집자) 1명: 콘텐츠 작성/수정/발행 담당
- 방문자: 독자, 잠재 구매자, 저자, 서점/유통 관계자

### 3.2 사용자 페르소나(Personas)
- **독자 A:** 신간/이벤트 소식을 빠르게 확인하고 구매 링크로 이동
- **독자 B:** 특정 저자 작품을 모아보고 싶음
- **저자/관계자 C:** 출판사 소개와 출간 이력 확인, 연락 채널 필요

## 5. 기능 요구사항(Functional Requirements)

### 5.1 공통(헤더/푸터)
- 헤더: 로고, 주요 메뉴, 검색(옵션), CTA(예: “신간 보기”)
- 푸터: 상호/사업자정보(선택), 이메일, SNS, 저작권, 개인정보처리방침 링크

**수용 기준**
- 모든 페이지에서 헤더/푸터 노출
- 모바일에서 햄버거 메뉴 정상 동작
- **(추가) 관리자 공지 팝업/배너**: 사이트 접속 시 최상단 배너 또는 팝업으로 중요 공지(휴무, 신간) 노출 가능

---

### 5.2 홈
- 최신 신간 섹션(최대 N권)
- 최신 소식 섹션(최대 N개)
- 출판사 소개 요약(미션/가치)
- “도서/소식”으로 이동하는 CTA

**수용 기준**
- CMS에서 등록한 최신 콘텐츠가 정렬 기준(발행일/출간일)대로 노출

---

### 5.3 도서(목록)
- 필수: 표지, 제목, 저자, 출간일(또는 발행일), 카테고리(옵션)
- 필터/정렬(1차 권장: **단순화**)
  - 정렬: 최신순(기본) 만 제공해도 초기엔 충분
  - 필터: 카테고리(선택)
- 페이지네이션(택1): 초기 도서 수가 적으면 더보기(Load More)도 무방하나, SEO 고려 시 페이지네이션 권장

**수용 기준**
- 도서가 100권 이상이어도 성능 문제 없이 탐색 가능(페이지네이션 권장)

---

### 5.4 도서(상세)
- 필수 콘텐츠
  - 제목, 표지, 저자(복수 가능), **역자(선택)**, **원제(선택)**, 소개(리치 텍스트), 출간일, ISBN(선택)
  - **시리즈 정보(선택)**: 시리즈명 + 권수
  - **미리보기 링크(선택)**: PDF 또는 서점 미리보기 URL
  - 외부서점 구매 링크 버튼 목록(예: 교보/리디/YES24/알라딘)
- 관련 콘텐츠(옵션)
  - 같은 저자 다른 책
  - 관련 소식(신간 발표 글 등)

**구매 링크 요구사항**
- 링크는 새 탭 열기 권장
- 링크 클릭 추적(Analytics 이벤트) 지원

**수용 기준**
- 구매 링크가 최소 1개 이상 등록되지 않으면 “구매처 준비중” 상태 표시

---

### 5.5 저자(목록)
- 저자 카드: 이름, 프로필 이미지(선택), 한 줄 소개(선택)
- 정렬: 이름순(기본), 최신 출간 순(옵션)

**수용 기준**
- 저자 클릭 시 상세 페이지 이동

---

### 5.6 저자(상세)
- 필수: 이름, 소개, 사진(선택), 외부 링크(선택)
- 자동 연결: 해당 저자의 도서 목록(대표도서부터 또는 최신순)

**수용 기준**
- 저자 페이지에서 도서 클릭 → 도서 상세로 이동

---

### 5.7 소식(목록/상세)
- 콘텐츠 유형(택1)
  - 단일 “소식” 타입에 태그/카테고리(신간/공지/이벤트/칼럼)
  - 또는 유형별 별도 타입(추천은 단일 타입 + 분류)
- 목록: 제목, 대표 이미지(선택), 요약, 발행일
- 상세: 본문(리치 텍스트), 이미지, 관련 도서(선택), 태그

**수용 기준**
- OG(공유 미리보기) 이미지/제목/요약이 정상 표시(SEO 요구사항 참조)

---

### 5.8 문의
- 최소 구성(권장)
  - 이메일, SNS/오픈채팅 링크(선택)
  - 문의 폼(선택): 이름/이메일/내용
- 문의 폼 선택 시: 스팸 방지(간단한 captcha 또는 honeypot)

**수용 기준**
- 문의 채널이 명확히 제공되고 모바일에서도 접근 가능

---

## 6. CMS 운영 요구사항(콘텐츠 관리)

### 6.1 CMS 도입 원칙
- 운영자는 CMS의 관리자 UI로만 작업
- 사이트에는 관리자 로그인/글쓰기 기능을 직접 만들지 않음(초기 범위)

### 6.2 콘텐츠 모델(권장)

#### Book
- `title` (필수)
- `slug` (필수, 고유)
- `coverImage` (필수)
- `description` (필수, 리치 텍스트)
- `publishDate` (필수 권장)
- `isbn` (선택)
- `translator` (선택: 역자)
- `originalTitle` (선택: 원제)
- `series` (선택: { name: string, volume: number })
- `previewLink` (선택: 미리보기 URL)
- `format` (선택: ebook/paper/both)
- `categories` (선택, 다중)
- `authors` (필수, 참조/관계)
- `buyLinks` (필수 권장: 배열)
  - `storeName` (예: 교보문고/리디/YES24/알라딘)
  - `url`

#### Author
- `name` (필수)
- `slug` (필수, 고유)
- `photo` (선택)
- `bio` (선택/권장)
- `links` (선택: 홈페이지/SNS 등)

#### Post (News)
- `title` (필수)
- `slug` (필수, 고유)
- `featuredImage` (선택)
- `excerpt` (선택/권장)
- `body` (필수, 리치 텍스트)
- `publishedAt` (필수 권장)
- `type` (필수 권장: 신간/공지/이벤트/칼럼)
- `relatedBooks` (선택, 참조)

### 6.3 콘텐츠 워크플로(권장)
- Draft(초안) → Review(검토) → Published(발행) 단순 단계(1인 운영이면 Draft/Published만으로도 충분)
- 발행 시 사이트 반영:
  - (A) CSR 방식이면 즉시 반영
  - (B) SSG/프리렌더 방식이면 **웹훅으로 자동 빌드/배포 트리거**

## 7. 비기능 요구사항(Non-functional Requirements)

### 7.1 성능
- LCP(대표 콘텐츠 표시) 개선을 위해 이미지 최적화(적절한 사이즈/포맷)
- 도서/소식 목록은 페이지네이션 기반(권장)

**수용 기준**
- 모바일 4G 환경에서도 주요 페이지가 체감상 빠르게 로딩(이미지 지연 로딩 포함)

### 7.2 SEO/공유(필수)
- 페이지별 메타(제목/설명) 제공
- Open Graph/Twitter 카드 지원(도서/소식 상세)
- `sitemap.xml`, `robots.txt` 제공
- Canonical URL 설정(가능하면)

**수용 기준**
- 카카오톡/트위터/디스코드 공유 시 제목/요약/이미지 정상 노출

### 7.3 접근성/디자인
- 기본 접근성 준수(명도 대비, 키보드 포커스, 대체 텍스트)
- 모바일 우선 반응형 레이아웃

### 7.4 보안
- 사이트 자체는 정적 배포로 서버 공격면 최소화
- CMS 계정은 강한 비밀번호 + 2FA 권장
- 문의 폼 사용 시 스팸 방지 적용

### 7.5 로깅/모니터링(권장)
- Analytics(예: GA4) 도입
- 이벤트: 구매 링크 클릭, 도서 상세 조회, 소식 상세 조회

## 8. 기술 아키텍처(구축 방향)

### 8.1 프론트엔드
- Vite + React
- 라우팅: `react-router-dom` (권장)
- 상태/데이터: 단순 fetch + 캐시(선택: React Query 등)

### 8.2 데이터 제공 방식 (확정: Git 기반 CMS - Keystatic)
- **Keystatic**
  - **Code-First**: TypeScript/JS 코드로 스키마 정의 (오타 방지, DX 우수)
  - **Local Mode**: 로컬에서 즉시 글 작성 및 파일 저장 지원 (`/content/books/*.mdoc` 등)
  - **Modern UI**: Notion과 유사한 직관적인 에디터 제공
  - 콘텐츠는 Git 저장소에 저장됨 (무료, 협업 용이)

**권장 결론:** 1인 출판사를 위한 최적의 DX/UX를 제공하는 **Keystatic + SSG** 확정

### 8.3 배포
- 정적 호스팅: Netlify / Cloudflare Pages / Vercel(정적) 중 택1
- 도메인 연결 + HTTPS

## 9. 페이지별 요구 데이터(요약)

- 홈: 최신 Book N개, 최신 Post N개
- 도서 목록: Book 리스트(페이지네이션), 카테고리 옵션
- 도서 상세: Book 1개 + authors + buyLinks + 관련 Post(옵션)
- 저자 목록: Author 리스트
- 저자 상세: Author 1개 + 해당 Author의 Book 리스트
- 소식 목록/상세: Post 리스트/상세 + 관련 Book(옵션)

- **SEO 부족 리스크(CSR만 사용 시):** SSG/프리렌더 도입 또는 공유용 메타 생성 전략 필요

## 11. 범위(스코프) 정의

### 11.1 MVP 범위(1차 런칭)
- 홈 / 도서 목록·상세 / 저자 목록·상세 / 소식 목록·상세 / 소개 / 문의(기본)
- CMS 콘텐츠 모델 3종(Book/Author/Post)
- 구매 링크 버튼(외부 서점) + 클릭 이벤트 트래킹(선택)
- SEO 기본(메타/OG, sitemap)

### 11.2 2차 확장(선택)
- 사이트 내 검색(도서/저자/소식 통합)
- 뉴스레터 구독(외부 서비스 연동)
- 보도자료(다운로드) 섹션
- 다국어(영문 페이지)

---

## 12. 수용 기준(Release Acceptance Criteria)

- 운영자가 CMS에서 **도서/저자/소식**을 등록·수정·발행할 수 있다.
- 도서 상세에서 **외부 서점 구매 링크**가 명확하게 노출되고 정상 이동한다.
- 저자 상세에서 해당 저자의 도서가 자동으로 연결되어 노출된다.
- 소식 글이 목록/상세에서 정상 노출되며 공유 미리보기가 정상이다.
- 모바일 화면에서 메뉴/목록/상세가 깨지지 않는다.

---

## 13. 일정(예시)

- 0주차: 요구사항 확정(이 PRD 확정) + CMS 선택
- 1주차: IA/디자인 와이어프레임 + CMS 스키마 확정
- 2주차: 프론트 페이지 구현(목록/상세) + CMS 연동
- 3주차: SEO/OG/배포 자동화 + QA + 런칭

> 실제 일정은 콘텐츠 양(도서 수, 기존 글 이관 여부)과 디자인 작업량에 따라 조정.

## 14. 오픈 질문(추후 결정 필요)
- CMS 후보: **Keystatic** 확정 (뛰어난 UI, 로컬 작업 편의성)
- 도서 카테고리/시리즈 체계(필수인지)
- 문의 폼을 넣을지(넣는다면 어떤 채널로 수신할지)
- “신간”을 별도 메뉴로 둘지(또는 소식/도서에서 자동 노출로 충분한지)

## 15. UI/UX 요구사항(최소 포함 권장)

> PRD에 “UI/UX를 얼마나 넣어야 하냐”는 팀 규모/협업 방식에 따라 다르지만, **개발·검수 기준이 될 최소 요건**은 PRD에 넣는 게 일반적입니다.  
> (상세한 화면 설계는 별도 문서/피그마로 분리해도 OK)

### 15.1 공통 UI 원칙
- 모바일 우선(320px~) 반응형, 터치 영역 44px 이상 권장
- 텍스트 가독성 우선(본문 15~17px 권장), 과도한 장식/애니메이션 지양
- 이미지(표지) 비율/규격 통일(예: 3:4) + 목록 썸네일 자동 크롭 규칙
- 접근성: 이미지 `alt` 필수(표지/인물), 키보드 포커스 표시, 대비 준수

### 15.2 주요 사용자 플로우(최소)
1) 홈 → 신간 클릭 → 도서 상세 → 외부서점 링크 클릭  
2) 저자 목록 → 저자 상세 → 해당 저자 도서 목록 탐색  
3) 소식 목록 → 소식 상세 → 관련 도서로 이동(선택)

### 15.3 페이지별 UI 구성(권장 체크리스트)
- 홈: 신간 카드 리스트(표지/제목/저자), 최신 소식 리스트(제목/날짜/요약)
- 도서 목록: 필터(카테고리/포맷), 정렬(최신/제목), 페이지네이션
- 도서 상세:
  - 상단: 표지 + 제목/저자/출간일
  - 본문: 소개(리치텍스트)
  - 구매 영역: 서점 버튼(브랜드 로고는 선택, 텍스트 버튼도 OK)
- 저자 상세: 프로필 + 소개 + 저서 리스트
- 소식 상세: 대표 이미지(선택) + 본문 + 태그 + 관련 도서(선택)

### 15.4 디자인 산출물(선택)
- (권장) 와이어프레임 6종: 홈/도서목록/도서상세/저자목록/저자상세/소식상세
- (있으면 좋음) 컴포넌트 목록: 카드, 버튼, 태그, 페이지네이션, 헤더/푸터
- (있으면 좋음) 디자인 토큰: 폰트 크기, 간격, 라운드, 그림자, 컬러(브랜드 컬러 1~2개)
