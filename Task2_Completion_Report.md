# Task 2 수정 완료 보고서 (Data Loading Stabilization)

## 수정 목적
홈페이지와 주요 컴포넌트에서 데이터가 없거나 형식이 올바르지 않을 때 발생하던 화면 깨짐(White Screen) 및 "Invalid Date" 표시 오류를 해결하기 위함입니다.

## 수정 내용

### 1. 도서 카드 날짜 표시 오류 수정 (`BookCard.tsx`)
- **문제**: 날짜 데이터(`publishDate`)가 없거나 형식이 잘못된 경우 화면에 "Invalid Date"라고 표시됨.
- **해결**:
  - `Date` 객체 변환 시 유효성 검사(`!isNaN`) 추가
  - 데이터 오류 시 기본값 "**발행일 미상**" 표시
  - 저자 데이터가 문자열 배열(Slug)로 들어올 경우에도 깨지지 않도록 타입 처리 추가

### 2. 뉴스 카드 및 UI 방어 코드 (`NewsCard.tsx`, `Home.tsx`)
- **문제**: 뉴스 날짜 역시 데이터가 불안정할 경우 오류 발생 가능성 있음.
- **해결**: `BookCard`와 동일한 방식의 날짜 파싱 방어 로직 적용.

### 3. 소개 페이지 크래시 수정 (`About.tsx`) - *선행 완료*
- **문제**: 데이터 파일(`about.mdoc`)이 없거나 내용을 불러오지 못하면 페이지 전체가 하얗게 변함.
- **해결**: 데이터가 `null`일 경우 "데이터를 불러올 수 없습니다." 메시지를 표시하도록 방어 코드 추가 및 기본 파일 생성.

## 결과
- 홈페이지의 도서 목록에서 더 이상 "Invalid Date"가 표시되지 않습니다.
- 데이터 로딩 실패 시에도 애플리케이션이 멈추지 않고 적절한 대체 UI를 보여줍니다.
