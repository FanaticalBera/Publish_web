# 도서출판 동틀녘 웹사이트 현황 분석 보고서

**작성일:** 2025-12-22  
**프로젝트:** Publish_web (React + TypeScript + Vite + Keystatic)

---

## 📋 프로젝트 개요

### 현재 구현된 기능
- ✅ **콘텐츠 관리 시스템 (CMS)**: Keystatic 기반 로컬 파일 관리
- ✅ **주요 페이지**: 홈, 도서 목록/상세, 작가 목록/상세, 소식, 자료실, 소개, 문의, 개인정보처리방침
- ✅ **반응형 디자인**: 모바일/태블릿/데스크톱 대응
- ✅ **SEO**: react-helmet-async 활용한 메타태그 관리
- ✅ **디자인 시스템**: Tailwind CSS + 커스텀 테마 (동틀녘 브랜드 컬러)

### 기술 스택
- React 18 + TypeScript
- Vite (빌드 도구)
- React Router v7
- Keystatic CMS
- Tailwind CSS + Radix UI

## 🔴 1. 수정이 필요한 사항

### 1.1 프로젝트 메타데이터
> **우선순위: 중**

**문제점:**
- `package.json`의 프로젝트명이 여전히 `"temp_project"`로 되어 있음
- `README.md`가 Vite 기본 템플릿 내용 그대로 남아있음

**권장 수정:**
- `package.json` → `"name": "dongtle-nyeok-publisher"`
- `README.md` → 프로젝트 소개, 설치 방법, 개발 가이드로 교체

---

### 1.2 접근성 (Accessibility) 개선
> **우선순위: 높음**

**문제점:**
- 이미지 대체 텍스트(alt) 누락 가능성
- 키보드 네비게이션 포커스 스타일 명시적 정의 부족
- ARIA 레이블 미흡

**권장 수정:**
- 모든 `<img>` 태그에 의미있는 `alt` 속성 추가
- 포커스 가시성 강화 (`:focus-visible` 스타일)
- 모바일 드로어, 버튼 등에 적절한 `aria-label` 추가

---

### 1.3 성능 최적화
> **우선순위: 중**

**문제점:**
- 이미지 최적화 누락 (WebP, 지연 로딩)
- 코드 스플리팅 미적용
- 폰트 로딩 최적화 필요

**권장 수정:**
- 이미지 지연 로딩(`loading="lazy"`) 적용
- React.lazy()로 페이지별 코드 스플리팅
- 폰트 preload 또는 font-display 전략 개선

---

### 1.4 에러 처리 강화
> **우선순위: 중**

**문제점:**
- 전역 에러 바운더리 미구현
- 404 페이지가 단순 텍스트만 표시
- 데이터 로딩 실패 시 사용자 피드백 부족

**권장 수정:**
- React Error Boundary 컴포넌트 추가
- 404 페이지 디자인 개선 (홈으로 돌아가기 버튼 등)
- 로딩 실패 시 재시도 기능 추가

---

### 1.5 환경변수 관리
> **우선순위: 낮음**

**문제점:**
- Vite 서버 포트가 하드코딩 (`port: 5176`)
- GA4 측정 ID 등이 설정 파일이 아닌 CMS 데이터에 의존

**권장 수정:**
- `.env` 파일로 환경변수 분리

## 🟡 2. 필요한 기능

### 2.1 검색 기능
> **우선순위: 높음**

**필요성:**  
사용자가 도서, 작가, 소식 등을 빠르게 찾을 수 있도록 검색 바 구현 필요

**구현 범위:**
- 헤더에 검색 아이콘/입력창
- 도서명, 작가명, 카테고리 기반 필터링
- 검색 결과 페이지 또는 모달

---

### 2.2 다국어 지원 (i18n)
> **우선순위: 중**

**필요성:**  
출판사가 해외 독자 대상 도서를 출간할 경우 영어/일본어 등 다국어 지원 필요

**구현 방안:**
- `react-i18next` 또는 `next-intl` 활용
- Keystatic 다국어 콘텐츠 스키마 확장

---

### 2.3 뉴스레터 구독 기능
> **우선순위: 중**

**필요성:**  
신간 소식, 이벤트 정보를 정기적으로 받고자 하는 독자를 위한 이메일 구독

**구현 방안:**
- 이메일 입력 폼 (Footer 또는 전용 페이지)
- Mailchimp, SendGrid, 또는 자체 백엔드 연동

---

### 2.4 도서 필터링 & 정렬
> **우선순위: 높음**

**필요성:**  
현재 도서 목록 페이지에 카테고리별 필터, 출간일/제목 정렬 기능 부재

**구현 범위:**
- 카테고리, 시리즈별 필터
- 신간순, 가나다순 정렬
- URL 쿼리스트링으로 상태 유지

---

### 2.5 소셜 공유 버튼
> **우선순위: 중**

**필요성:**  
도서 상세 페이지, 뉴스 상세 페이지에서 SNS 공유 기능 필요

**구현 범위:**
- 카카오톡, 페이스북, 트위터 공유 버튼
- Open Graph 메타태그 동적 생성 (이미 부분 구현됨)

---

### 2.6 관리자 대시보드 개선
> **우선순위: 낮음**

**필요성:**  
Keystatic 기본 UI는 좋지만, 통계/분석 기능 부재

**구현 방안:**
- 도서별 조회수, 인기 순위 표시
- Google Analytics 데이터 연동 위젯

---

### 2.7 폼 스팸 방지
> **우선순위: 중**

**필요성:**  
Contact 페이지나 뉴스레터 구독에 스팸 방지 장치 필요

**구현 방안:**
- Google reCAPTCHA v3
- Honeypot 필드

## 🟢 3. 있으면 좋을 기능 (Nice-to-Have)

### 3.1 독자 리뷰 & 평점 시스템
**설명:**  
도서 상세 페이지에 독자들이 별점과 리뷰를 남길 수 있는 기능

**참고:**
- Disqus, Utterances 같은 댓글 시스템 연동
- 또는 자체 백엔드 구축

---

### 3.2 도서 미리보기 뷰어
**설명:**  
PDF 미리보기를 iframe이 아닌 전용 뷰어로 개선

**참고:**
- PDF.js 활용
- 페이지 넘김, 확대/축소 기능

---

### 3.3 작가 인터뷰 & 칼럼 섹션
**설명:**  
News 타입에 이미 '칼럼'이 있지만, 별도의 '작가의 말' 섹션으로 분리

**장점:**
- 독자와 작가 간 친밀감 형성
- SEO 효과 (양질의 콘텐츠)

### 3.4 이벤트 & 북토크 캘린더
**설명:**  
출판사 주최 행사, 북토크 일정을 캘린더 형식으로 표시

**참고:**
- `react-big-calendar` 또는 `fullcalendar`

---

### 3.5 독자 커뮤니티 (포럼/게시판)
**설명:**  
독자들이 자유롭게 의견을 나누는 공간

**참고:**
- Discourse, phpBB 같은 기존 솔루션
- 또는 댓글 시스템 확장

---

### 3.6 전자책 뷰어 내장
**설명:**  
리디북스처럼 사이트 내에서 직접 전자책을 읽을 수 있는 기능

**참고:**
- EPUB.js 활용
- DRM 고려 필요

---

### 3.7 위시리스트 & 장바구니
**설명:**  
사용자가 관심 도서를 담아두고, 외부 서점으로 일괄 구매 연결

**참고:**
- LocalStorage 또는 사용자 계정 연동
- 교보문고 API 등 제휴 필요

---

### 3.8 다크 모드
**설명:**  
CSS에 이미 다크 모드 변수가 정의되어 있으나, 토글 UI 미구현

**구현:**
- Header에 다크 모드 토글 버튼
- `localStorage`로 사용자 선택 저장

---

### 3.9 애니메이션 & 인터랙션 강화
**설명:**  
스크롤 애니메이션, 패럴랙스 효과 등으로 사용자 경험 향상

**참고:**
- Framer Motion, GSAP

---

### 3.10 PWA (Progressive Web App) 지원
**설명:**  
모바일에서 앱처럼 설치 가능하도록 PWA 기능 추가

**구현:**
- Service Worker 등록
- manifest.json 생성

---

### 3.11 RSS 피드
**설명:**  
신간 소식이나 뉴스를 RSS로 제공하여 독자들이 구독 가능하게

**참고:**
- `/rss.xml` 엔드포인트 생성

---

### 3.12 A/B 테스팅 & 분석 도구
**설명:**  
두 가지 디자인이나 카피를 테스트하여 전환율 개선
